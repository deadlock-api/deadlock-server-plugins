The `DeadworksManaged.Api.Trace` static class exposes VPhys2 ray and
shape casts to plugins. Currently marked on the wiki as *"not used in
any example plugin"* (`[[plugin-api-surface]]:74`) and otherwise
undocumented. Worth a dedicated entity page.

## Three entrypoints

```csharp
// Low-level: raw shape cast
static void TraceShape(Vector3 start, Vector3 end, Ray_t ray, CTraceFilter filter, ref CGameTrace trace);

// Mid-level: constructs filter/ray for you, but you still need mask enums
static void SimpleTrace(Vector3 start, Vector3 end,
    RayType_t rayKind, RnQueryObjectSet objectQuery,
    MaskTrace interactWith, MaskTrace interactExclude, MaskTrace interactAs,
    CollisionGroup collision, ref CGameTrace trace,
    CBaseEntity? filterEntity = null, CBaseEntity? filterSecondEntity = null);

// + angles variant that projects pitch/yaw forward instead of taking end point:
static void SimpleTraceAngles(Vector3 start, Vector3 angles, … float maxDistance = 8192f);

// High-level: simple line ray with sensible defaults
static TraceResult Ray(Vector3 start, Vector3 end,
    MaskTrace mask = MaskTrace.Solid | MaskTrace.Hitbox,
    CBaseEntity? ignore = null);
```

## TraceResult (simplified) vs CGameTrace (full 192-byte struct)

`Trace.Ray` returns `TraceResult { HitPosition, Fraction, DidHit, Trace }`
where `Trace` is the full `CGameTrace` for when you need more than the
convenience fields.

`CGameTrace` exposes `HitEntity` (computes `CBaseEntity?` from `pEntity`),
`Distance`, `Direction`, `StartInSolid`, plus
`HitEntityByDesignerName(name, out outEntity, NameMatchType = StartsWith)`
for classname-filtered tests.

## Shape variety (`Ray_t` union)

`Ray_t` is an explicit-layout union (48 bytes) carrying one of
`LineTrace`, `SphereTrace` (center + radius), `HullTrace` (mins/maxs
AABB), `CapsuleTrace` (two centers + radius), or `MeshTrace`
(convex mesh). `Type` field at offset 0x28 indicates which variant is
active.

`Ray_t.CreateLine(start)` is the common factory; shape kinds need
`new Ray_t { Type = RayType_t.Sphere, Sphere = ... }` or the `Init`
overloads on the struct.

## Silent no-op when not ready

`Trace.TraceShape` returns immediately if
`NativeInterop.TraceShapeFn == 0` or `PhysicsQueryPtr` is null
(`TraceSystem.cs:7-9`). Safe to call from `OnLoad` — will just be a
no-op until the physics query system is wired up during map start.
`Trace.Ray` returns a default result (`Fraction=1, DidHit=false,
HitPosition=end`) in the not-ready path.

## Filter vtable gotcha

`CTraceFilter` embeds a function-pointer vtable (2 slots: destructor +
`ShouldHitEntity`) allocated once in `CTraceFilterVTable`'s static
constructor. A filter constructed via `default(CTraceFilter)` or
`Marshal.PtrToStructure` won't have `_vtable` set — `EnsureValid()` is
called internally before dispatch to paper over this, but directly
calling the native trace with a zero-vtable filter would crash. The
`bool checkIgnoredEntities` ctor arg selects `Simple` (always-hit)
vs. `WithEntityFilter` (honours `EntityIdsToIgnore`).

## Practical defaults

- Default mask for `Ray`: `MaskTrace.Solid | MaskTrace.Hitbox` —
  world geometry + hitboxes.
- `ObjectSetMask = RnQueryObjectSet.All` — query all object sets
  unless filtered.
- `CollisionGroup = CollisionGroup.Always` — no collision-group
  filtering by default.

## Interop surface

Backed by `NativeInterop.TraceShapeFn` (function pointer) and
`NativeInterop.PhysicsQueryPtr` (double-ptr into the game's physics
query system). Set by native bootstrap; not directly plugin-accessible.

## Potential uses

None currently — but the API supports, e.g., LOS-checking before
particle spawns, identifying what a player is looking at, auto-aim
prototypes, geometry-snap teleports. Plugin-surface documentation
should signal "this works" since the silent-no-op + full-featured
profile is not obvious from filenames alone.
